# Heading

見出しを表示するコンポーネント。文書の構造（アウトライン）を表す意味的なタグと、見た目のサイズを分離して指定できる。

<Canvas of={HeadingStories.Default} />

## as と size の関係

- `as`: `h1`〜`h6`。文書のアウトライン上の意味的なレベルを表す。スクリーンリーダーの見出しジャンプ等はこの値を基準にする
- `size`: `sm` / `md` / `lg` / `xl` / `2xl` / `3xl`。見た目の大きさ
- `size` を省略すると `as` の値から自動的にマッピングされる（`h1`→`3xl`, `h2`→`2xl`, `h3`→`xl`, `h4`→`lg`, `h5`→`md`, `h6`→`sm`）
- 明示的に `size` を指定すると、`as` に関わらずそのサイズで上書きできる（例: 文書構造上は `h3` だが、見た目は控えめにしたい場合）

<Canvas of={HeadingStories.AllLevels} />

## Do's / Don'ts

- 見た目のサイズを変えたいだけの場合は `size` を使う。`as` を「大きく見せたいから」という理由だけで変えない（文書のアウトラインが崩れる）
- ページ内で `h1` は基本的に1つ。セクションの見出しには `h2` 以降を使う
- 視覚的なレイアウト上の都合で見出しレベルをスキップしない（`h2` の次にいきなり `h4` を使う、など）。読み上げ順とインデントの手がかりが失われる
- 見出しではない強調テキストには `Heading` を使わない（`Text` 等、本文用のコンポーネントを検討する）

## アクセシビリティ

- ネイティブの `h1`〜`h6` 要素をそのまま使っているため、スクリーンリーダーの見出しジャンプ機能に自然に対応する
- 見た目のサイズ（`size`）だけを理由に意味的なレベル（`as`）を選ばないこと。文字の大小は装飾であり、文書構造の代わりにはならない
