# Heading

見出しを表示するコンポーネント。文書の構造（アウトライン）を表す意味的なタグと、見た目のサイズを分離して指定できる。

## as と size の関係

- `as`: `h1`〜`h6`。文書のアウトライン上の意味的なレベルを表す。スクリーンリーダーの見出しジャンプ等はこの値を基準にする
- `size`: `sm` / `md` / `lg` / `xl` / `2xl` / `3xl`。見た目の大きさ
- `size` を省略すると `as` の値から自動的にマッピングされる（`h1`→`3xl`, `h2`→`2xl`, `h3`→`xl`, `h4`→`lg`, `h5`→`md`, `h6`→`sm`）
- 明示的に `size` を指定すると、`as` に関わらずそのサイズで上書きできる（例: 文書構造上は `h3` だが、見た目は控えめにしたい場合）

## color

- `color`: `default` / `white` / `secondary` / `disabled` / `link` / `inherit`。デザイントークンで定義された色のみ指定できる（任意のCSS色文字列は指定できない）
- 通常の見出しは `default`（省略時のデフォルト）を使う。控えめにしたい場合は `secondary`、暗い背景の上では `white` を使う
- `disabled` / `link` は主に他コンポーネント（フォームのラベルやリンク付きの見出し等）との組み合わせを想定した値で、単体のHeadingで使う場面は少ない
- `inherit` は親要素の`color`をそのまま継承したい場合に使う

<Canvas of={HeadingStories.AllColors} />

## marginBottom

- `marginBottom`: `none`（デフォルト） / `xs` / `sm` / `md` / `lg` / `xl` / `2xl`。デザイントークン（spacing）で定義された余白のみ指定できる（任意のCSS値・px指定はできない）
- Headingのベーススタイルは`margin: 0`（リセット済み）のため、見出しの下に余白を空けたい場合は`marginBottom`を明示的に指定する

## Do's / Don'ts

- 見た目のサイズを変えたいだけの場合は `size` を使う。`as` を「大きく見せたいから」という理由だけで変えない（文書のアウトラインが崩れる）
- ページ内で `h1` は基本的に1つ。セクションの見出しには `h2` 以降を使う
- 視覚的なレイアウト上の都合で見出しレベルをスキップしない（`h2` の次にいきなり `h4` を使う、など）。読み上げ順とインデントの手がかりが失われる
- 見出しではない強調テキストには `Heading` を使わない（`Text` 等、本文用のコンポーネントを検討する）
- `color` はデザイントークンで定義された値からのみ選ぶ。インラインスタイルや任意の16進数カラーで見出しの色を上書きしない（デザインの一貫性が崩れる）
- 余白を調整したい場合は`marginBottom`（デザイントークン）を使う。`className`で任意のCSSマージンを直接上書きしない

## アクセシビリティ

- ネイティブの `h1`〜`h6` 要素をそのまま使っているため、スクリーンリーダーの見出しジャンプ機能に自然に対応する
- 見た目のサイズ（`size`）だけを理由に意味的なレベル（`as`）を選ばないこと。文字の大小は装飾であり、文書構造の代わりにはならない
