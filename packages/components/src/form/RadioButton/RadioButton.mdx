# RadioButton

複数の選択肢から1つだけを選択させるための、[RadioGroup](../RadioGroup/RadioGroup.mdx)を構成する個々の選択肢コンポーネント。Ark UIの`RadioGroup.Item`（`RadioGroup.Root`/`Label`/`Item`/`ItemControl`/`ItemText`/`ItemHiddenInput`の複合コンポーネントのうちの1つ）を、Checkboxと同じ方針（ADR 0012）で単一コンポーネントに完全ラップして実装している。

## RadioGroupの子孫としてのみ動作する

RadioButtonはArk UIの`RadioGroup.Item`相当で、選択状態の共有に`RadioGroupContext`を使うため、[RadioGroup](../RadioGroup/RadioGroup.mdx)（Ark UIの`RadioGroup.Root`）の子孫としてレンダリングされることを前提とする。Checkboxとは異なり、RadioButton単体を独立して使うことは想定していない。

## label（選択肢のラベル）

- `label`: `string`。必須
- 選択肢のテキストとして`RadioGroup.ItemText`に描画される

## value（選択肢の識別子）

- `value`: `string`。必須
- RadioGroup側の`value`/`defaultValue`と比較され、一致する選択肢が選択状態になる

## disabled

ネイティブの`disabled`属性をそのまま使う。個別の選択肢だけを無効化したい場合に使う（グループ全体を無効化したい場合は[RadioGroup](../RadioGroup/RadioGroup.mdx)の`disabled`を使う）。

<Canvas of={RadioButtonStories.Disabled} />

## className / style / children は受け付けない

RadioButtonは`className`・`style`・`children`propを受け付けない（ADR 0012）。ラベルは`label`（string）で指定する。

## Do's / Don'ts

- 複数の選択肢から1つを選ばせたい場合は、RadioButtonを並べた[RadioGroup](../RadioGroup/RadioGroup.mdx)を使う。RadioButtonを単体で（RadioGroupの外で）使わない
- 選択中の視覚表現（中央のドット）はCSSの`::after`疑似要素で描画しているため、`ItemControl`に子要素を追加しても反映されない

## アクセシビリティ

- Ark UIの`RadioGroup.ItemHiddenInput`（ネイティブの`<input type="radio">`）を使うため、キーボード操作（Tab/矢印キー）やスクリーンリーダーでのロール認識は追加の実装なしにブラウザ標準の挙動に従う
- `disabled`時はネイティブの`disabled`属性を使う。フォーカス移動の対象から除外され、クリックイベントも発火しない
