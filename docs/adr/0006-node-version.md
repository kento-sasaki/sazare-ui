# ADR 0006: Node.js バージョン

## コンテキスト

要件14番で Node のバージョン管理に mise を採用することは決まっているが、固定するバージョンは未定だった。要件5番「採用するライブラリは可能な限り最新のバージョンを使いたい」がこの決定にも適用されるかを確認する必要があった。

## 決定ドライバー(評価軸)

1. npm 公開するライブラリのビルド・CI基盤としての安定性
2. `engines` フィールドを通じた消費者側（mabiki）との互換性
3. 最新性

## 検討した選択肢

（2026-07-10 時点で nodejs.org の公開情報を実測）

| 選択肢 | 強み | 弱み |
|---|---|---|
| Node 24（最新LTS "Krypton"、2026-06-23時点でのlatest） | LTSのため安定性が高く、`engines` で要求しても消費者側との互換性リスクが低い | 「最新」という意味では26系に劣る |
| Node 26（最新Current、非LTS） | 文字通り最新 | 非LTSのためCIやconsumer環境との互換性リスクがある |

## 決定

**Node 24（LTS "Krypton"）** を mise で固定する。要件5番の「最新版を使いたい」は主にライブラリ選定（依存パッケージ）に適用し、ビルド基盤である Node 自体は安定性を優先してLTSを選ぶ。

## 影響・懸念（未解決事項）

- Node の LTS はおよそ半年〜1年周期で更新されるため、次期LTSへの追従タイミングは別途判断する。
