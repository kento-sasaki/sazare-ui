# ADR 0010: Typographyコンポーネントの意味的レベルと見た目サイズの分離

## コンテキスト

Heading（Issue #27）の実装にあたり、見出しレベル（h1〜h6）と見た目のサイズをどう関連付けるかを決める必要があった。この決定は次のText（Issue #28）を含む今後のtypographyカテゴリ全体（およびサイズという概念を持つ他コンポーネント）にも及ぶ命名規則になるため、Heading単体の実装詳細ではなく、デザインシステム全体の規約として決めておく。

## 決定ドライバー（評価軸）

1. 文書のアウトライン（スクリーンリーダーの見出しジャンプ、SEO等）を壊さずに見た目の柔軟性を確保できること
2. Text等、見出しレベルという概念を持たない他のtypographyコンポーネントともサイズ命名を共有できること
3. 実装・APIのシンプルさ

## 検討した選択肢

| 選択肢 | 強み | 弱み |
|---|---|---|
| `level`のみ（意味的タグと見た目サイズが常に連動） | propが1つでシンプル | 文書構造と見た目を独立して制御できない。「文書構造上はh3だが見た目は控えめにしたい」に対応できない |
| `as`（意味的タグ）と`size`（見た目、Tシャツサイズ）を分離 | 文書構造と見た目を独立して制御できる。`size`の命名をText等の他typographyコンポーネントと共有できる | propが2つになる |

## 決定

**`as`（意味的な見出しレベル）と`size`（見た目のサイズ）を分離する。**

- `as`: `h1`〜`h6`。文書のアウトライン上の意味的なレベルを表す
- `size`: `sm` / `md` / `lg` / `xl` / `2xl` / `3xl`（Tシャツサイズ命名）。見た目の大きさを表す
- `size`を省略した場合は`as`の値から自動的にマッピングする（`h1`→`3xl`, `h2`→`2xl`, `h3`→`xl`, `h4`→`lg`, `h5`→`md`, `h6`→`sm`）。素直に使えば意味的レベルと見た目が一致する
- 明示的に`size`を指定すれば`as`に関わらず上書きできる

この設計はRadix ThemesのHeadingコンポーネント等でも採用されているパターンで、Ark UI/Park UIにはHeading専用プリミティブが存在しないためプレーンなセマンティックHTML + Panda CSS recipeで実装する（ADR 0001）。

## 影響・懸念（未解決事項）

- 次のText（Issue #28）はheadingレベルを持たないため`as`は不要だが、`size`のTシャツサイズ命名とスケール値（sm/md/lg/xl/2xl/3xl）は踏襲する
- font-sizeスケールの具体的な値（`packages/tokens/src/typography.ts`）はv0.1のdraft運用（要件16番）における初期値であり、後から調整可能
