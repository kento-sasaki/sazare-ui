# sazare-ui - Claude Code Guide

## Claude Code 動作指針

**不明点があれば必ず質問すること**

ユーザーからの指示について、以下のような場合は作業を進める前に必ず `AskUserQuestion` ツールを使って確認すること：

- 指示の意図や目的が曖昧な場合
- 複数の解釈が可能な場合
- 具体的な仕様（ファイル名、パス、形式など）が不明な場合
- 既存の設計方針やルールと矛盾する可能性がある場合
- 影響範囲が大きい変更を行う場合

**推測で進めない。確認してから進める。**

## 過去の経緯の参照

実装の際に過去の意思決定の経緯を把握したい場合は `docs/adr` を参照すること。設計方針や技術選定の背景・理由が ADR（Architecture Decision Record）としてまとめられている。
